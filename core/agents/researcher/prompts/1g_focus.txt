사용자의 질문 목록과 키워드 클러스터 목록이 주어진다. 각 클러스터에 대해 '이 키워드로 검색해서 얻은 데이터가 사용자 질문에 답하는 블로그 글을 쓸 때 직접 인용하거나 근거로 사용되는가'를 판단하라.

## 판단 기준
focus: true → 해당 클러스터의 검색 데이터(상위 콘텐츠, 검색량, SERP 등)가 블로그 글의 본문 작성에 직접 활용됨
focus: false → 아래 중 하나라도 해당:
- 질문 주제와 다른 산업/맥락 (예: 'ERP 외주' 질문에 'EMP')
- 검색 의도가 정보 탐색이 아닌 내비게이션/사전/번역 (예: 'X 위키', 'X 영어로')
- 질문과 완전히 무관한 키워드가 자동완성으로 혼입 (예: 외주개발 질문에 '이사청소')
- 주제 범위가 너무 넓어 질문에 특화된 데이터를 제공하지 못함 (예: 'ERP' 단독, '앱개발' 단독처럼 범용 키워드만 포함된 클러스터)
- 질문에 간접적으로만 관련되고, 이 클러스터 없이도 글을 쓸 수 있음

## few-shot 예시
질문: '외주 개발 업체 선정 기준은?'
- "외주 개발 비용" → true (비용은 선정 기준의 핵심)
- "외주 개발 계약서" → true (계약은 선정 시 중요 요소)
- "이사청소 후기" → false (완전히 다른 주제)
- "ERP 위키" → false (사전 정의 검색, 콘텐츠 작성에 불필요)
- "앱개발" → false (질문은 '업체 선정'인데 '앱개발' 범용 키워드는 간접적)
- "프로젝트 팀 구성" → false (프로젝트 관리 일반론, 외주 업체 선정과 간접적)

## 규칙 (반드시 준수)
- 반드시 전체 클러스터의 30% 이상을 false로 판정하라. 이것은 절대 규칙이다.
- focus 비율 목표: 50~70%. 모든 클러스터가 필요해 보여도 가장 간접적인 것부터 false로 돌려라.
- 확신이 없으면 false로 판정하라 (보수적 판단)
- reason 필드에 판단 근거를 한 문장으로 작성하라
- JSON 배열만 반환하라.
