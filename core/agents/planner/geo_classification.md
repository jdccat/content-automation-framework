# GEO 유형 분류

> 6단계에서 참조한다.
> 콘텐츠가 AI 검색 엔진에서 인용되기 유리한 구조를 설계하기 위한 분류 기준.

---

## 유형 정의

### 정의형

"~란 무엇인가"에 답하는 콘텐츠. AI가 개념 설명이나 요약 응답을 생성할 때 인용한다.

**구조 패턴**
- 도입부 2~3문장에 핵심 정의를 명확하게 제시
- 이후 구성요소, 종류, 작동 원리 등으로 확장
- 표, 비교표, 구조화된 목록 활용

**인용 유리 조건**
- 첫 문단에서 질문에 대한 직접 답변이 완결
- 전문 용어에 대한 쉬운 풀이가 포함
- 스키마 마크업(FAQ, HowTo) 적용 가능

**예시 질문**
- "ERP란 무엇인가요"
- "IT 아웃소싱 뜻"
- "애자일 개발 방법론이란"

---

### 비교형

"A와 B 중 어떤 것이 나은가"에 답하는 콘텐츠. AI가 선택지를 제시하거나 추천할 때 인용한다.

**구조 패턴**
- 비교 기준 프레임을 먼저 제시
- 기준별로 항목을 나란히 비교 (표 필수)
- 상황별 추천으로 마무리 ("~한 경우 A, ~한 경우 B")

**인용 유리 조건**
- 비교 기준이 명시적이고 일관됨
- 일방적 추천이 아닌 조건부 추천
- 비교 대상의 장단점이 균형 있게 서술

**예시 질문**
- "외주 vs 인하우스 개발 비교"
- "앱 개발 업체 선정 기준"
- "워터폴 vs 애자일 차이"

---

### 문제해결형

"~할 때 어떻게 해야 하나"에 답하는 콘텐츠. AI가 절차를 안내하거나 해결책을 제시할 때 인용한다.

**구조 패턴**
- 문제 상황을 구체적으로 특정
- 단계별 해결 절차를 순서대로 나열
- 각 단계에 주의사항 또는 체크포인트 포함
- 실패 케이스나 흔한 실수를 별도 섹션으로

**인용 유리 조건**
- 단계가 번호로 명확하게 구분
- 각 단계가 독립적으로 인용 가능한 단위
- 구체적 수치, 기간, 비용 등 실행 정보 포함

**예시 질문**
- "외주 개발 프로젝트 진행 절차"
- "개발 견적서 검토 방법"
- "외주 계약서 작성 시 주의사항"

---

## 에이전트 처리 규칙

6단계에서 각 콘텐츠의 하위 질문 텍스트를 보고 유형을 배정한다. 하나의 콘텐츠가 복수 유형에 걸치면 지배적 유형 하나를 선택하고, H2 구조 내에 부차적 유형의 패턴을 부분 삽입한다.

예: "외주 업체 선정 기준과 절차" → 지배적 유형: 비교형, H2 중 하나에 문제해결형(절차) 패턴 삽입.

---

## 클라이언트 확장 시 변경 포인트

- **유형 추가**: 산업에 따라 "사례형"(성공/실패 사례 중심), "데이터형"(자체 데이터 기반 인사이트) 등을 추가할 수 있다.
- **구조 패턴 조정**: AI 검색 엔진의 인용 패턴이 산업마다 다를 수 있으므로, 인용 유리 조건을 클라이언트 도메인에 맞게 튜닝한다.
- **3개 기본 유형은 유지한다.** 정의형, 비교형, 문제해결형은 모든 클라이언트 공통.
