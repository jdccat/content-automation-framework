# 사용자 입력 템플릿 — 위시켓

> 플래너 에이전트는 이 파일에 정의된 항목만 사용자 입력으로 받는다.
> 템플릿에 없는 항목을 임의로 요구하지 않는다.

---

## 입력 항목

### 1. 질문 의도

사용자가 이번 월간 콘텐츠를 통해 답하려는 핵심 의도. **1개 이상 선택 가능**.
이 의도가 1단계 파생 질문 정렬의 기준이다.

| 선택지 | 정의 | 정렬 효과 |
|--------|------|-----------|
| 정보 탐색 | 특정 주제에 대해 체계적으로 알고 싶은 상태. 아직 선택지를 좁히지 않았다. | 탐색 경로의 자연 순서를 그대로 따른다. 넓은 질문 → 좁은 질문 순서. |
| 비교 판단 | 선택지가 있고, 어느 쪽이 나은지 기준을 잡고 싶은 상태. | 비교형 질문을 상위로 올린다. 기준 제시 → 항목별 비교 → 판단 보조 순서. |
| 추천 | 선택지를 좁혀서 구체적인 답을 받고 싶은 상태. | 선택지를 좁혀주는 질문을 상위로 올린다. 조건 확인 → 후보 축소 → 최종 제안 순서. |

**에이전트 처리 규칙**
- 1단계에서 리서처가 확장한 하위 질문 리스트를 정렬할 때, 여기서 선택된 의도에 따라 정렬 우선순위가 달라진다.
- 의도가 복수 선택된 경우, 선택된 의도들의 정렬 효과를 조합한다. 예를 들어 "비교 판단 + 추천"이면 비교형 질문과 선택지 축소형 질문을 모두 상위로 올리되, 입력 순서상 먼저 선택된 의도에 약간의 우선을 둔다.
- 질문 의도는 정렬에만 관여한다. 개별 콘텐츠의 톤과 H2 구조는 `content_direction.md`의 GEO 유형 × 퍼널 태그 조합이 결정한다.

---

### 2. 질문 형태

쿼리 팬아웃 전의 큰 단위 질문. **1개 이상 복수 입력 가능**.

각 질문은 사용자가 AI 검색 엔진에 직접 입력하는 형태와 동일하다. 리서처는 각 질문에서 하위 질문을 확장하고, 플래너는 각 질문을 하나의 **카테고리**로 취급한다.

**형식 규칙**
- 자연어 질문 형태로 입력한다. 키워드 나열이 아니라 완결된 질문이어야 한다.
- 질문 하나가 카테고리 하나가 된다. 질문이 3개면 카테고리도 3개다.
- 서로 다른 검색 의도를 담은 질문을 입력한다. 같은 의도의 표현만 다른 질문은 리서처 단계에서 하나로 병합된다.

**카테고리 배분 규칙**
- 기획 문서에 포함되는 콘텐츠는 카테고리별로 균등 배분한다.
- 월간 발행 수 ÷ 카테고리 수 = 카테고리당 콘텐츠 수.
- 나누어떨어지지 않으면 탐색 경로가 더 깊은 카테고리에 1건씩 추가 배정한다.
- 예시: 월 12건 / 카테고리 3개 → 카테고리당 4건.
- 예시: 월 12건 / 카테고리 4개 → 카테고리당 3건.

---

### 3. 콘텐츠 방향성

이번 달 콘텐츠의 전략적 강조점. **1개 이상 선택 가능**.
방향성은 월간 콘텐츠 포트폴리오의 구성 비중을 결정하며, 4단계 콘텐츠 선발과 5단계 퍼널 균형 조정에서 참조한다. 개별 콘텐츠의 H2 구조, CTA, 톤은 `content_direction.md`의 GEO 유형 × 퍼널 태그 조합이 결정한다.

| 선택지 | 정의 | 선발 효과 |
|--------|------|-----------|
| 카테고리 포지셔닝 | 특정 키워드 카테고리에서 위시켓을 대표 브랜드로 인식시키는 데 집중한다. | 인지 퍼널 콘텐츠와 정의형 GEO 유형의 비중을 높인다. 해당 카테고리의 검색량 상위 키워드를 우선 선발한다. |
| 문제 인식 확산 | 고객이 아직 인지하지 못한 문제를 드러내서 수요를 만든다. | 인지~고려 퍼널 콘텐츠를 우선 선발한다. 커뮤니티 질문 패턴에서 발견된 잠재 문제 키워드의 선발 순위를 올린다. |
| 판단 기준 제시 | 고객이 비교·선택하는 과정에서 신뢰할 수 있는 기준을 제공한다. | 고려 퍼널 콘텐츠와 비교형 GEO 유형의 비중을 높인다. 경쟁 난이도가 중간 이하인 비교 키워드를 우선 선발한다. |
| 실행 가이드 | 고객이 실제로 행동에 옮길 수 있는 구체적 절차를 안내한다. | 고려~전환 퍼널 콘텐츠와 문제해결형 GEO 유형의 비중을 높인다. 절차·방법 키워드를 우선 선발한다. |

**에이전트 처리 규칙**
- 4단계 콘텐츠 선발 시, 선택된 방향성이 지시하는 퍼널·GEO 유형 비중을 반영하여 선발 우선순위를 조정한다.
- 방향성이 복수 선택된 경우, 각 방향성의 선발 효과를 조합한다. 입력 순서상 먼저 선택된 방향성에 약간의 우선을 둔다.
- 5단계 퍼널 균형 점검 시, 선택된 방향성이 의도하는 퍼널 분포와 실제 선발 결과의 퍼널 분포를 비교한다. 괴리가 크면 코멘트를 남긴다.
- 방향성은 선발과 균형 조정에만 관여한다. 선발된 개별 콘텐츠의 H2 구조, CTA, 톤은 `content_direction.md`가 결정한다.

## 입력 예시

```
질문 의도: 비교 판단

질문 형태:
1. ERP 외주 개발 업체를 고를 때 가장 중요하게 봐야 할 기준은 무엇인가요?
2. 앱 개발 견적이 업체마다 다른 이유는 무엇이며 어떤 기준으로 판단해야 하나요?
3. 외주 개발 프로젝트를 진행할 때 가장 자주 발생하는 문제는 무엇이며, 어떻게 해결할 수 있나요?

콘텐츠 방향성: 판단 기준 제시
```

**이 예시의 에이전트 해석**

- 카테고리 3개가 생성된다. 월 12건 기준 카테고리당 4건 배정.
- 질문 의도가 "비교 판단"이므로 1단계에서 비교형 하위 질문이 탐색 경로 상위로 정렬된다.
- 콘텐츠 방향성이 "판단 기준 제시"이므로 4단계 선발 시 고려 퍼널 콘텐츠와 비교형 GEO 유형의 비중을 높인다. 경쟁 난이도가 중간 이하인 비교 키워드가 우선 선발된다.
- 선발된 개별 콘텐츠의 H2 구조, CTA, 톤은 `content_direction.md`의 GEO 유형 × 퍼널 태그 조합에 따라 결정된다.

## 입력 검증 규칙

플래너는 입력을 받으면 아래 조건을 확인한다. 하나라도 미충족이면 Slack으로 재입력을 요청한다.

| 조건 | 실패 시 메시지 |
|------|-------------|
| 질문 의도가 3개 선택지 중 1개 이상인가 | "질문 의도를 정보 탐색 / 비교 판단 / 추천 중 1개 이상 선택해 주세요." |
| 질문 형태가 1개 이상 입력되었는가 | "질문을 최소 1개 이상 입력해 주세요." |
| 질문 형태가 키워드가 아닌 완결된 질문인가 | "키워드가 아닌 질문 형태로 입력해 주세요. 예: '외주 개발' → '외주 개발 진행 과정을 알려줘'" |
| 콘텐츠 방향성이 4개 선택지 중 1개 이상인가 | "콘텐츠 방향성을 카테고리 포지셔닝 / 문제 인식 확산 / 판단 기준 제시 / 실행 가이드 중 1개 이상 선택해 주세요." |

## 역할 분담 요약

| 입력 항목 | 관여 범위 | 관여하지 않는 범위 |
|----------|---------|----------------|
| 질문 의도 | 1단계 하위 질문 정렬 순서 | 개별 콘텐츠의 톤, H2 구조, CTA |
| 질문 형태 | 카테고리 생성, 리서처 시드 | — |
| 콘텐츠 방향성 | 4단계 선발 비중, 5단계 퍼널 균형 점검 | 개별 콘텐츠의 H2 구조, CTA, 톤 |

개별 콘텐츠의 H2 구조, CTA, 톤은 모두 `content_direction.md`의 GEO 유형 × 퍼널 태그 조합이 결정한다.

## 다른 클라이언트 확장 시 변경 포인트

이 파일은 클라이언트별로 커스터마이징된다. 변경 가능한 항목은 다음과 같다.

- **질문 의도 선택지**: 추가(예: "벤치마크 탐색") 또는 제거(예: "추천" 삭제) 가능. 각 선택지의 정렬 효과를 함께 정의해야 한다.
- **콘텐츠 방향성 선택지**: 산업과 전략에 따라 교체 가능. 각 선택지의 선발 효과를 함께 정의해야 한다.
- **선택 제한**: 필요 시 특정 항목을 단일 선택으로 제한할 수 있다. 제한 시 검증 규칙도 함께 업데이트한다.
- **입력 검증 규칙**: 선택지 변경에 따라 검증 메시지도 업데이트한다.
- **질문 형태의 형식 규칙**: 변경하지 않는다. 자연어 질문 형태 입력은 모든 클라이언트 공통이다.
