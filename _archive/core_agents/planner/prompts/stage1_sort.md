# Stage 1 — 카테고리 매핑 및 연계 질문 생성

## 역할

리서처가 수집한 클러스터 데이터를 분석하여, 각 클러스터를 사용자 입력 카테고리에 매핑하고, 해당 카테고리의 독자에게 최적화된 연계 질문을 생성한다.

## 입력

- `categories`: 사용자 입력 질문 목록. 각각이 하나의 독립 카테고리다.
- `per_category_target`: 각 카테고리당 필요한 파생 질문 최소 개수.
- `clusters`: 리서처 클러스터 목록. 클러스터당 다음 데이터가 제공된다.
  - `cluster_id`: 클러스터 식별자
  - `representative_keyword`: 클러스터의 대표 키워드
  - `paa_questions`: Google PAA / 네이버 연관검색에서 수집된 실제 사용자 질문
  - `keywords`: 클러스터에 속한 실제 검색 키워드 목록
  - `naver_top_titles`: 네이버 상위 콘텐츠 제목 (해당 키워드를 검색하면 실제 노출되는 글)
  - `google_top_titles`: 구글 상위 콘텐츠 제목

## 처리 규칙

### 1단계: 클러스터 의미 파악

각 클러스터에 대해 다음 데이터를 종합적으로 읽고 "이 클러스터가 실제로 다루는 주제"를 파악한다.

- `paa_questions`: 사용자가 실제로 물어보는 질문 패턴
- `naver_top_titles` / `google_top_titles`: 이 키워드로 검색했을 때 실제 노출되는 콘텐츠 제목 (가장 중요한 신호)
- `keywords`: 클러스터 내 세부 키워드

### 2단계: 카테고리 매핑

파악한 주제를 바탕으로 가장 연관된 카테고리 1개에 배정한다.

- 하나의 클러스터는 반드시 하나의 카테고리에만 배정된다.
- 어느 카테고리와도 연관성이 없으면 `"unassigned"`로 표기한다.

### 3단계: 연계 질문 생성

카테고리에 배정된 클러스터에 대해, 아래 기준으로 `derived_question`을 생성한다.

- 이 클러스터의 주제와 카테고리 질문의 사용자 의도가 만나는 지점에서 질문을 만든다.
- 카테고리 질문을 읽는 사람이 다음에 자연스럽게 찾아볼 만한 구체적인 하위 질문이어야 한다.
- `paa_questions`에 이미 적합한 질문이 있으면 그대로 사용해도 된다.
- 없으면 `naver_top_titles`, `google_top_titles`에서 반복되는 관점을 반영해 새로 만든다.
- 반드시 "~인가요?", "~까요?", "~나요?", "~할까요?" 등 자연스러운 질문 형태로 끝내야 한다.

### 3-1단계: 카테고리별 질문 수 조정

각 카테고리에 배정된 클러스터 수가 `per_category_target`보다 적으면,
기존 클러스터에서 서로 다른 각도의 질문을 최대 2개까지 생성한다.

- 두 번째 질문은 첫 번째 질문과 중복되지 않는 다른 하위 주제여야 한다.
- 클러스터 수가 충분한 카테고리는 클러스터당 질문 1개만 생성한다.

### 4단계: 매핑 이유 기록 (수치 포함 필수)

`mapping_reason`에 다음을 반드시 포함한다.

- **신호 수치**: 데이터 소스별로 전체 항목 수 대비 관련 신호 수를 표기한다.
  - 예: "naver_top_titles 5개 중 3개에서 '업체 선정 기준' 패턴 확인"
  - 예: "paa_questions 7개 중 4개가 비용·견적 주제"
- **결정 신호**: 최종 카테고리 배정에 가장 큰 영향을 준 구체적인 제목 또는 질문을 1개 이상 직접 인용한다.
- **연계 질문 출처**: paa에서 선택했는지, naver/google 타이틀에서 발견했는지, 새로 생성했는지 명시한다.

## 출력 형식

반드시 아래 JSON 객체만 반환한다. 설명, 마크다운 코드 블록 없이 JSON만 반환한다.

{
  "mappings": [
    {
      "cluster_id": "<cluster_id>",
      "category": "<카테고리 원문 또는 unassigned>",
      "derived_questions": ["<질문1>", "<질문2(선택)>"],
      "mapping_reason": "<이 클러스터를 이 카테고리에 배정하고 이 질문을 생성한 이유>"
    }
  ]
}

### 출력 제약

- 입력의 모든 `cluster_id`가 반드시 `mappings`에 1개씩 포함되어야 한다.
- `category` 값은 입력으로 받은 `categories` 배열의 원문을 그대로 사용하거나 `"unassigned"`만 허용한다. 원문을 수정하거나 요약하지 않는다.
- `derived_questions`는 1~2개 배열이며, 각 항목은 반드시 질문 형태로 끝나야 한다.
- 두 번째 질문은 첫 번째와 다른 하위 주제를 다뤄야 하며, 카테고리당 배정된 클러스터 수가 `per_category_target`에 미달할 때만 추가한다.
