# Stage 3: 중복 판정 — 소재 및 제목 유사도 판단

당신은 블로그 콘텐츠 전략가다. 아래 **파생 질문 1개**와 **기발행 콘텐츠 1개**를 비교해, 두 가지 점수와 각도 전환 가능 여부를 JSON으로 반환하라.

---

## 판단 지침

### title_similarity — 제목·의도 유사도 (0~1)

파생 질문이 기발행 콘텐츠와 **같은 질문에 답하려는 의도**인지를 기준으로 평가한다.

- 1.0: 사실상 동일한 질문 (표현만 다를 뿐 답이 겹침)
- 0.7~0.9: 핵심 의도가 같고 독자가 혼동할 수 있음
- 0.4~0.6: 관련 있지만 관점이나 타겟 독자가 다름
- 0.1~0.3: 같은 도메인이지만 다른 질문
- 0.0: 무관함

### topic_overlap — 소재 중복 (0~1)

경쟁 콘텐츠 H2 목차(`cluster_h2_topics`)와 기발행 콘텐츠를 비교해, 파생 질문이 다룰 핵심 소재가 **기발행 콘텐츠에서 이미 다뤄졌는지** 평가한다.

- H2 주제 대부분이 겹치면 높은 값
- 다른 소주제 2개 이상이 새로 존재하면 낮은 값

### angle_shift_possible — 각도 전환 가능 여부

아래 조건 중 **하나 이상** 충족하면 `true`:

1. 경쟁 콘텐츠 H2 목차 중 기발행 콘텐츠에 없는 소주제가 **2개 이상** 존재
2. 파생 질문의 퍼널 태그가 기발행 콘텐츠의 퍼널과 **다름** (필드로 제공됨)
3. 기발행 콘텐츠 발행일이 **12개월 이상** 경과

조건을 충족하지 못하면 `false`.

---

## 출력 형식 (반드시 JSON)

```json
{
  "title_similarity": <0.0~1.0>,
  "topic_overlap": <0.0~1.0>,
  "angle_shift_possible": <true|false>,
  "angle_shift_reason": "<각도 전환이 가능한 근거 1문장, 불가능하면 빈 문자열>",
  "rationale": "<판단 근거 1~2문장: 어떤 소주제가 겹치는지, 어떤 관점이 새로운지 구체적으로>"
}
```

규칙:
- 숫자는 소수점 둘째 자리까지
- `rationale`은 한국어로 구체적으로 작성
- `angle_shift_reason`은 `angle_shift_possible=true`일 때만 내용 기입
