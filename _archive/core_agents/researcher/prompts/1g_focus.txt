사용자의 질문 목록과 키워드 클러스터 목록이 주어진다. 각 클러스터에 대해 '이 키워드로 검색해서 얻은 데이터가 사용자 질문에 답하는 블로그 글을 쓸 때 직접 인용하거나 근거로 사용되는가'를 판단하라.

## 핵심 원칙: 질문 범위 일치 (최우선)
질문이 특정 도메인/범위를 명시하면 (예: "ERP 외주", "앱 개발", "SaaS 도입"), 해당 도메인 한정어가 클러스터 키워드에 포함되어야 한다. 도메인 한정어 없는 범용 클러스터는 기본 false.
- 질문 "ERP 외주 개발 업체 선택 기준" → "erp 외주 비용" = true, "외주 개발 비용" = false
- 질문 "앱 개발 업체 비교" → "앱 개발 업체 추천" = true, "개발 업체" = false

## 판단 기준
focus: true → 아래 조건을 **모두** 충족:
1. 질문의 도메인 한정어가 클러스터 키워드에 포함됨 (또는 직접적 동의어)
2. 검색 데이터가 블로그 글 본문에 직접 활용됨 (인용·통계·사례로)
3. 이 데이터 없이는 글의 완성도가 명확히 떨어짐

focus: false → 아래 중 하나라도 해당:
- 질문의 도메인 한정어(예: ERP, 앱, SaaS)가 클러스터 대표+멤버에 부재하고 범용적
- 질문의 핵심 행위(예: 선택, 비교)와 맞지 않는 후속 단계 주제 (예: 관리, 운영)
- 질문 주제와 다른 산업/맥락
- 검색 의도가 정보 탐색이 아닌 내비게이션/사전/번역
- 질문과 무관한 키워드 혼입
- 범위가 넓어 질문 특화 데이터를 제공 못함
- 간접적으로만 관련되고, 이 클러스터 없이도 글을 쓸 수 있음

## few-shot 예시
질문: 'ERP 외주 개발 업체를 고를 때 가장 중요하게 봐야 할 기준은?'
- "erp 외주 개발 업체 비교" → true (ERP 외주 업체 비교는 선정 핵심)
- "erp 외주 가격" → true (ERP 외주 비용은 선정 기준)
- "erp 외주 개발 사례" → true (ERP 사례는 업체 판단 근거)
- "erp 외주 개발 방법" → true (ERP 외주 진행 방법은 기준 이해에 필수)
- "외주 개발 비용" → false (범용 외주, ERP 도메인 한정어 없음)
- "개발 업체" → false (범용, ERP·외주 한정어 없음)
- "외주 개발 계약서" → false (범용이고, 선택 '기준'이 아닌 선정 이후 절차)
- "외주 개발 업체 관리 팁" → false (선정 이후 관리 단계, 선택 기준과 간접적)
- "앱 개발 업체" → false (다른 도메인)
- "외주 개발 플랫폼" → false (플랫폼 자체는 ERP 업체 선택 기준과 무관)

## 규칙 (반드시 준수)
- 반드시 전체 클러스터의 40% 이상을 false로 판정하라. 이것은 절대 규칙이다.
- focus 비율 목표: 30~50%. 질문 범위에 가장 밀접한 것만 남겨라.
- 도메인 한정어 없는 범용 클러스터는 기본 false로 시작하라.
- 확신이 없으면 false로 판정하라 (보수적 판단).
- reason 필드에 판단 근거를 한 문장으로 작성하라.
- JSON 배열만 반환하라.
