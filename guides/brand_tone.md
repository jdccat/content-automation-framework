# 브랜드 톤 가이드

> designer가 타이틀, H2 헤딩, CTA, 발행 목적을 설계할 때 참조한다.
> 퍼널별 톤은 `content_direction.md`의 구조 지침과 함께 적용한다.

---

## 브랜드 보이스

**Professional + Clear + Practical**

- 전문가처럼 말하지만
- 이해하기 쉽게 설명하고
- 실무에 도움이 되어야 한다

---

## 핵심 독자

- IT 비전문가 기업 담당자 (기술 전문가가 아닌 비즈니스 의사결정자)
- 내부 개발 인력이 부족한 기업 실무자
- AI 도입 및 AX를 고민하는 기업 담당자
- 개발 업체, IT 인력을 찾아야 하는 기획자·운영 담당자

**설계 원칙**: 기술 설명은 쉽게 풀고, 기업 상황 중심으로 설명하며, 실무 의사결정에 도움이 되는 정보를 제공한다.

---

## 퍼널별 톤

### 인지

설명적, 중립, 전문 용어를 쓰되 괄호로 풀어줌. 자사 서비스를 본문에서 언급하지 않는다.

### 고려

분석적, 근거 중심, 양쪽 다 제시한 뒤 조건부 추천. "일반적으로"보다 "A 조건이면 ~, B 조건이면 ~".

### 전환

실행 중심, 구체적, 숫자와 기간 포함. 확신을 주되 과장하지 않음.

---

## 글 스타일

### 문제 중심 출발

콘텐츠는 항상 기업의 문제 상황에서 시작한다.

- ❌ "AI는 기업 경쟁력을 높이는 중요한 기술입니다."
- ✅ "많은 기업이 AI 도입을 고민하지만 어디서부터 시작해야 할지 몰라 프로젝트가 멈추는 경우가 많습니다."

### 실무 기준 제시

이론 설명보다 실제 의사결정에 필요한 기준을 제공한다: 개발 업체 선택 기준, 프로젝트 비용 구조, AI 도입 방법, 외주 개발 리스크 관리.

### 과장 없는 설득

- ❌ 금지: 최고의 플랫폼, 완벽한 솔루션, 반드시 성공
- ✅ 권장: 실제 사례, 의사결정 기준, 현실적인 설명

### 자연스러운 위시켓 연결

위시켓은 콘텐츠의 주제가 아니라 해결 방법 중 하나로 등장해야 한다.

- ❌ "그래서 위시켓을 사용하면 됩니다."
- ✅ "검증된 개발사를 빠르게 비교할 수 있는 플랫폼을 활용하는 방법도 있습니다."

---

## 브랜드 귀속

핵심 판단이 '위시켓의 판단'으로 남아야 한다.

- "위시켓 기준으로 보면" 또는 동등한 표현이 포함되어야 한다
- 결론 문장에서 판단의 주체가 명확히 위시켓이어야 한다
- 브랜드명이 빠져도 의미가 통하는 일반론이 되지 않아야 한다

**위시켓만 할 수 있는 말**:
- 실제 프로젝트를 다수 중개한 경험에서 나온 판단
- 반복적으로 관찰된 실패/성공 패턴
- 플랫폼을 써보지 않으면 알 수 없는 구조적 한계

---

## 톤 공통 규칙

- **문체**: ~입니다/~합니다 경어체
- **전문성 수위**: 클라이언트(발주자) 대상이면 기술 용어 최소화, 파트너(수행자) 대상이면 실무 용어 사용
- **위시켓 언급**: 본문에서 직접 홍보하지 않는다. CTA 영역에서만 안내
- **근거 제시**: 숫자, 사례, 조건부 추천 선호. "일반적으로 ~합니다" 지양

---

## 지양/지향 톤

| 지양 | 지향 |
|------|------|
| 과장된 마케팅 문구 ("최고의 개발사를 만나보세요") | 실무 중심 설명 ("AI 챗봇을 도입하면 고객 문의 대응 시간을 줄일 수 있습니다") |
| 기술 중심 설명 ("AI 모델을 활용한 NLP 시스템") | 의사결정 기준 제공 ("개발 업체를 선택할 때는 다음 3가지를 확인해야 합니다") |
| 근거 없는 주장, 광고성 문장 | 데이터·사례 기반, 조건부 추천 |

---

## designer용 체크리스트

기획안 설계 시 아래 항목을 확인한다.

- [ ] 도입부가 독자의 문제/궁금증에서 시작하는가
- [ ] H2 헤딩이 과장 없이 실무 기준을 반영하는가
- [ ] 각 H2 섹션이 독립적으로 추출 가능한 답변 블록인가
- [ ] 각 H2 섹션에 표·리스트·수치가 배치될 수 있는 구조인가
- [ ] "위시켓만 할 수 있는 말"이 기획안에 반영될 여지가 있는가
- [ ] CTA가 본문 흐름의 논리적 종착지인가
- [ ] 발행 목적이 고객 여정(인지→등록→구축→운영→확장)에 정렬되는가

## 타이틀 작성 규칙

공식을 미리 정하지 않는다. 참고 예시와 평가 기준만 제시하고, 후보 생성 후 확률 추론으로 최종 선택한다.

**길이**: 25~35자 (구분자 포함 시 45자까지)

**참고 예시** (기발행 DB에서 추출):
- "개발 업체, 어떻게 골라야 할까? 현장에서 검증한 5가지 기준"
- "외주 개발, 어떤 리스크가 있을까? 자주 발생하는 4가지 문제와 해결법"
- "홈페이지 제작 완벽 가이드 : 비용, 기간, 과정 한눈에 정리"
- "앱 개발 업체 찾는 법 : 어디서, 어떻게 선택해야 할까"
- "키오스크 개발 가이드 : 비용·외주 업체 선택까지 총정리"
- "프리랜서 플랫폼 비교 2025｜특징·수수료·상황별 추천"

**생성 프로세스** (designer가 따를 절차):
1. 후보 3~4개를 자유롭게 생성
2. 각 후보에 "사용자가 이 타이틀을 선택할 확률(%)"을 추론
3. 가장 높은 확률의 타이틀 1개를 최종 선택
4. 나머지 후보도 `title_suggestions` 배열에 확률 내림차순으로 포함

**평가 기준** (추론 시 고려):
- 메인 키워드가 앞쪽에 자연스럽게 들어가는가
- 독자가 클릭하고 싶은 구체적 힌트(숫자, 질문, 해결 약속)가 있는가
- 기발행 DB 톤(과장 없이, 실무 중심)과 일관되는가
- update 콘텐츠라면 연도(2026) 표기가 있는가

**금지**: 최고의/완벽한/반드시 성공, 50자 초과

## H2 헤딩 작성 규칙

### 형식 규칙

- **길이**: 10~25자 (스캐너블)
- **개수**: 3~5개 (최적 4)
- **첫 H2 패턴** — 정의형: "~이란?", 비교형: 차이/이유 프레임, 문제해결형: 문제 유형 나열
- **중간 H2 패턴** — 리스트형: "핵심 기준 N가지", 스텝형: "1단계 — [행동]", 질문형: "어떻게 ~해야 할까?", **질문 기반형**: "~할 때 가장 중요한 기준은?"
- **마지막 H2 패턴** (CTA 연결) — 소프트 질문: "~이 궁금하다면?", 실행 안내: "~하는 방법", 체크리스트: "최종 체크리스트"
- **금지**: 이모지 남용, 30자 초과, H2에서 위시켓 직접 언급(CTA 전용)

### GEO 인용 설계 원칙

H2는 AI가 섹션 단위로 추출·인용하는 기본 단위다. 다음 3가지 원칙을 따른다.

- **독립 답변 블록**: 각 H2 섹션은 앞뒤 문맥 없이 단독으로 의미가 통해야 한다. AI는 섹션 단위로 추출·인용한다.
- **구조화 포맷 전제**: 섹션 내에 표·리스트·체크리스트가 들어갈 수 있도록 H2를 설계한다.
- **근거 세트 전제**: 사례·수치·조건이 붙을 수 있는 H2를 설계한다. 근거 없는 섹션은 AI 인용 후보에서 밀린다.

### DO/DON'T 예시

| DO | DON'T |
|----|-------|
| "외주 개발 업체 선정 핵심 기준 5가지" | "🔥 최고의 외주 개발 업체를 찾는 완벽한 방법!!" |
| "앱 개발 비용, 왜 업체마다 다를까?" | "앱 개발 비용에 대해 알아야 할 모든 것을 정리했습니다" |
| "검증된 파트너를 찾고 싶다면?" | "위시켓에서 파트너를 찾아보세요!" |
| "앱 개발 비용이 업체마다 다른 3가지 이유" (표·비교 배치 가능) | "앱 개발 비용에 대하여" (나열만 가능한 구조) |

## CTA 텍스트 작성 규칙

**기본 구조**: `[위시켓 기능/서비스]로 [독자 행동]해 보세요`

**퍼널별 강도**:
- **인지**: "~에 대해 더 알아보세요" / 관련 콘텐츠 안내. 1문장
- **고려**: "위시켓에서 [비교/검증]해 보세요". 1문장 30자 이내
- **전환**: "위시켓에서 프로젝트를 등록하면 [구체적 혜택]". 2문장까지 허용

**위시켓 언급**: CTA에서만 직접 언급. 본문 H2는 간접 표현

**수치 삽입**: "평균 25건의 제안", "검증된 파트너" 등 플랫폼 데이터 활용 권장

**금지**: "지금 바로 클릭", "무료 상담", 3문장 초과

### DO/DON'T 예시

| DO | DON'T |
|----|-------|
| "위시켓에서 검증된 개발사를 비교해 보세요" | "지금 바로 클릭하세요!" |
| "프로젝트를 등록하면 평균 25건의 제안을 받을 수 있습니다" | "무료 상담 신청하고 완벽한 솔루션을 만나보세요" |
| "외주 개발에 대해 더 알아보세요" | "위시켓은 최고의 플랫폼입니다. 지금 바로 가입하세요. 후회하지 않을 겁니다." |
