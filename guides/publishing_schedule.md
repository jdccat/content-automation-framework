# 발행 일정 가이드

> 4단계(선발 수량 결정)와 5단계(발행일 배정)에서 참조한다.
> 이 파일의 숫자가 월간 기획의 물리적 상한이다.

---

## 발행 파라미터

| 항목 | 값 | 비고 |
|------|---|------|
| 주간 발행 수 | 3회 | |
| 월간 발행 수 | 12건 | 주간 발행 수 × 4주 기준. 해당 월 실제 주 수와 무관하게 고정. |
| 발행 요일 | 월 · 수 · 금 | |
| 발행 시간 | 09:00 KST | CMS 예약 발행 기준 |

## 4단계 연결: 선발 수량 결정

월간 발행 수를 사용자 입력 카테고리 수로 나눠 카테고리당 배정 수를 결정한다.

```
카테고리당 배정 수 = 월간 발행 수 ÷ 카테고리 수
```

| 카테고리 수 | 카테고리당 배정 | 잔여 배분 |
|-----------|-------------|---------|
| 2 | 6건 | — |
| 3 | 4건 | — |
| 4 | 3건 | — |
| 5 | 2건 | 잔여 2건 → 탐색 경로가 더 깊은 카테고리에 1건씩 |

나누어떨어지지 않으면 탐색 경로가 더 깊은 카테고리 순으로 1건씩 추가 배정한다. 탐색 경로 깊이가 동일하면 검색량 합산이 높은 카테고리에 배정한다.

---

## 5단계 연결: 캘린더 배치 규칙

선정된 콘텐츠를 발행 요일에 배치한다. 배치 시 아래 규칙을 순서대로 적용한다. 상위 규칙이 하위 규칙보다 우선한다.

### 규칙 1. 퍼널 교차 배치

같은 퍼널의 콘텐츠가 연속 2회 이상 배치되지 않는 조합을 우선 제안한다.

```
월(인지) → 수(고려) → 금(전환) — 우선 제안
월(인지) → 수(인지) → 금(고려) — 가능하나 후순위
```

퍼널 분포상 연속 배치가 불가피한 경우에도 별도 경고 없이 그대로 배치한다.

### 규칙 2. 카테고리 순환 배치

같은 카테고리의 콘텐츠가 같은 주에 2건 이상 배치되지 않도록 한다. 카테고리 간 교차로 배치해서 독자가 매주 다양한 주제를 접하게 한다.

```
카테고리 3개(A, B, C) / 주 3회 발행 시:
1주차: 월(A) → 수(B) → 금(C)
2주차: 월(A) → 수(B) → 금(C)
```

카테고리 수가 발행 요일 수보다 적으면(예: 카테고리 2개, 주 3회) 불가피하게 같은 주에 같은 카테고리가 2건 배치된다. 이 경우 퍼널 교차 배치를 우선 적용한다.

### 규칙 3. 탐색 경로 순서 배치

같은 카테고리 내 콘텐츠는 1단계에서 정렬한 탐색 경로 순서대로 발행한다. 탐색 경로 앞쪽(넓은 질문)이 먼저, 뒤쪽(좁은 질문)이 나중에 발행된다.

```
카테고리 A 탐색 경로: A1 → A2 → A3 → A4
발행 순서: 1주차 A1, 2주차 A2, 3주차 A3, 4주차 A4
```

## 예외 처리

| 상황 | 대응 |
|------|------|
| 해당 월 공휴일이 발행 요일과 겹침 | 해당 건에 "공휴일 발행" 태그를 부여. 발행일 이동 없음. 발행 여부는 사용자가 판단. |
| 월간 발행 수 미달 (선발 콘텐츠 < 12건) | 대기 풀에서 추가 선발. 대기 풀도 부족하면 실제 선발 수로 캘린더를 구성하고 기획 문서에 미달 사유를 표기. |
| 긴급 콘텐츠 삽입 요청 | 해당 주 발행 수를 1건 추가(4건). 기존 콘텐츠는 밀어내지 않는다. |

## 클라이언트 확장 시 변경 포인트

- **발행 파라미터**: 주간 발행 수, 월간 발행 수, 발행 요일, 발행 시간 모두 클라이언트별 조정.
- **배치 규칙 우선순위**: 규칙 1~3의 순서는 유지하되, 클라이언트 요청에 따라 특정 규칙의 가중치를 조정할 수 있다. 예: "카테고리 순환보다 탐색 경로 순서를 우선해달라"는 요청 시 규칙 2와 3의 순서를 교체.
- **배치 규칙 자체는 변경하지 않는다.** 퍼널 교차, 카테고리 순환, 탐색 경로 순서의 3개 규칙 구조는 모든 클라이언트 공통.
