# 콘텐츠 방향 가이드

> 개별 콘텐츠에 배정된 퍼널 태그(funnel_criteria.md)와 GEO 유형(geo_classification.md)의 조합으로 H2 구조와 CTA를 결정한다.
> 톤 관련 지침은 `brand_tone.md`를 참조한다.

---

## 결정 구조

```
개별 콘텐츠
  ├─ GEO 유형 (정의형 / 비교형 / 문제해결형)  →  H2 구조 패턴
  ├─ 퍼널 태그 (인지 / 고려 / 전환)            →  CTA 강도 + 도입부 설계
  └─ 조합                                      →  콘텐츠 세부 구조
```

GEO 유형이 글의 **뼈대**(어떤 순서로 전개하는가)를 결정하고, 퍼널 태그가 **CTA 강도와 도입부 프레이밍**을 결정한다. 톤은 `brand_tone.md` 참조.

---

## GEO 유형별 H2 패턴

> `geo_classification.md`에서 유형 정의와 인용 유리 조건을 참조. 여기서는 실제 H2 흐름만 정의한다.

### 정의형

**H2 흐름**: 핵심 정의 (2~3문장) → 구성 요소 또는 유형 분류 → 활용 맥락 → 다음 탐색 안내

**위시켓 예시**:
- H2-1: ERP 외주 개발이란? (정의 + 자체 개발과의 차이)
- H2-2: ERP 외주의 주요 방식 3가지
- H2-3: 어떤 상황에서 외주가 적합한가
- H2-4: 다음 단계 안내

### 비교형

**H2 흐름**: 비교 프레임 제시 → 기준별 비교 → 조건부 추천 → 판단 보조

**위시켓 예시**:
- H2-1: 외주 vs 자체 개발 — 무엇을 기준으로 고를까
- H2-2: 비용 비교
- H2-3: 기간 비교
- H2-4: 품질 관리 비교
- H2-5: 우리 상황에 맞는 선택은?

### 문제해결형

**H2 흐름**: 구체적 문제 제시 → 단계별 해결 절차 (번호 매김) → 단계별 체크포인트 → 흔한 실수

**위시켓 예시**:
- H2-1: 시작 전 준비할 3가지
- H2-2: 1단계 — 요구사항 정의서 작성
- H2-3: 2단계 — 업체 후보 리스트업과 미팅
- H2-4: 3단계 — 견적 비교 및 계약
- H2-5: 자주 발생하는 실수와 예방법

## 퍼널별 CTA 강도와 도입부

> 퍼널별 톤은 `brand_tone.md`를 참조한다.

### 인지

독자가 개념이나 카테고리를 처음 탐색하는 상태. 아직 선택지를 좁히지 않았다.

**CTA 강도**: 약. 전환을 유도하지 않는다. 관련 콘텐츠 탐색 또는 가이드 다운로드 수준.
**도입부 설계**: 독자의 현재 궁금증을 그대로 반영한 질문으로 시작. "~란 무엇일까요?"

### 고려

독자가 선택지를 알고 있고, 비교하거나 기준을 세우는 상태.

**CTA 강도**: 중. 판단을 도와주는 도구(견적 비교, 체크리스트)로 자연스럽게 연결.
**도입부 설계**: 독자의 고민 상황을 구체적으로 묘사. "업체 3곳에서 견적을 받았는데 가격이 다 다르다면?"

### 전환

독자가 결정을 내렸거나 내리기 직전. 실행 정보가 필요한 상태.

**CTA 강도**: 강. 본문 흐름 안에서 서비스 이용 절차를 자연스럽게 안내. 프로젝트 등록, 상담 신청 등 직접 전환 액션.
**도입부 설계**: 바로 실행에 필요한 정보를 약속. "이 글을 따라하면 오늘 안에 프로젝트 등록까지 완료할 수 있습니다."

## 조합 가이드 (GEO 유형 × 퍼널)

에이전트는 아래 매트릭스에서 해당 셀의 지침을 따른다.

### 정의형 × 인지

가장 기본적인 카테고리 설명 콘텐츠. 검색 유입의 입구.
- H2: 정의형 패턴 그대로
- CTA: 관련 콘텐츠 링크
- 예: "IT 아웃소싱이란? 개념부터 유형까지 정리"

### 정의형 × 고려

개념은 알지만 자기 상황에 맞는지 확인하고 싶은 독자.
- H2: 정의형 패턴에서 "활용 맥락" 섹션을 강화. 상황별 적합성 판단 기준 추가.
- CTA: 자가 진단 체크리스트, 견적 비교
- 예: "ERP 외주 개발, 우리 회사에 맞는 방식은?"

### 정의형 × 전환

드문 조합. 정의형인데 전환 단계면, 독자가 용어 확인 차원에서 검색한 것일 수 있다.
- H2: 정의형 패턴을 짧게 처리하고, 마지막 섹션에서 실행 절차로 바로 연결.
- CTA: 프로젝트 등록, 상담 신청
- 코멘트: "이 조합은 콘텐츠 기획 의도를 재확인하세요. 정의형보다 문제해결형이 더 적합할 수 있습니다."

### 비교형 × 인지

아직 기준을 못 세웠지만, 선택지가 있다는 건 아는 상태.
- H2: 비교형 패턴에서 "비교 프레임 제시" 섹션을 두텁게. 기준 자체를 교육.
- CTA: 관련 콘텐츠, 가이드 다운로드
- 예: "외주 개발 업체, 어떤 기준으로 비교해야 할까?"

### 비교형 × 고려

가장 전형적인 고려 콘텐츠. 기준을 세우고 항목별로 비교.
- H2: 비교형 패턴 그대로
- CTA: 견적 비교, 업체 평가 도구
- 예: "앱 개발 견적이 업체마다 다른 이유와 판단 기준"

### 비교형 × 전환

비교를 마치고 최종 결정 직전. 의사결정을 확정시켜주는 콘텐츠.
- H2: 비교형 패턴에서 "조건부 추천" 섹션을 강화. 구체적 조건 → 명확한 추천.
- CTA: 프로젝트 등록, 상담 신청
- 예: "외주 개발 업체 최종 선택 전 반드시 확인할 3가지"

### 문제해결형 × 인지

문제를 인식하기 시작한 독자. 해결 방법보다 문제의 실체를 파악하고 싶은 상태.
- H2: 문제해결형 패턴에서 "구체적 문제 제시"와 "흔한 실수" 섹션을 강화. 절차는 개요 수준.
- CTA: 관련 콘텐츠, 자가 진단
- 예: "외주 개발 프로젝트에서 가장 자주 발생하는 문제 5가지"

### 문제해결형 × 고려

문제를 알고, 해결 방법을 비교하는 상태.
- H2: 문제해결형 패턴 그대로. 단계별 절차에 "이 단계에서 확인할 판단 기준" 추가.
- CTA: 체크리스트 다운로드, 견적 비교
- 예: "외주 개발 리스크를 줄이는 계약 전 체크리스트"

### 문제해결형 × 전환

해결 방법을 알고, 지금 바로 실행하려는 독자.
- H2: 문제해결형 패턴에서 절차를 가장 구체적으로. 숫자, 기간, 준비물 명시.
- CTA: 프로젝트 등록, 상담 신청
- 예: "외주 개발 프로젝트 등록부터 계약까지, 5단계 실행 가이드"

## CTA 매핑 — 위시켓

| CTA | 설명 | 적합 퍼널 |
|-----|------|---------|
| 관련 콘텐츠 탐색 | 같은 카테고리 또는 다음 단계 블로그 글 링크 | 인지 |
| 가이드 다운로드 | 체크리스트, 템플릿, 가이드북 PDF | 인지, 고려 |
| 견적 비교 | 여러 업체 견적을 나란히 비교하는 페이지 | 고려 |
| 프로젝트 등록 | 위시켓에 프로젝트를 올려 견적을 받는 행위 | 고려, 전환 |
| 상담 신청 | 위시켓 전문가와 1:1 상담 예약 | 전환 |
| 파트너 등록 | 위시켓 파트너(수행사/프리랜서)로 가입 | 파트너 대상 전용 |

**배치 규칙**
- 본문 CTA: 마지막 또는 마지막 직전 H2 섹션에 자연스럽게 삽입. 1회.
- 마무리 CTA: 글 말미에 1회.
- 총 최대 2회. 3회 이상 반복하지 않는다.
- 인지 퍼널 콘텐츠에서는 마무리 CTA만 1회.

## 클라이언트 확장 시 변경 포인트

- **CTA 매핑 테이블**: 전면 교체. 클라이언트별 전환 경로에 맞게 항목명, URL, 적합 퍼널 재정의.
- **톤 공통 규칙**: 문체, 전문성 수위, 브랜드 언급 방식을 교체.
- **조합 가이드의 예시**: 위시켓 도메인 예시를 클라이언트 도메인 예시로 교체.
- **GEO 유형별 H2 패턴과 퍼널별 톤 정의는 공통.** 변경하지 않는다.
